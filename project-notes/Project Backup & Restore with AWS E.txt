Project: Backup & Restore with AWS EBS & S3
📌 What is EBS?

EBS (Elastic Block Store) = A hard disk for your EC2 instance.

Just like your laptop has C: drive (Windows) or /dev/sda (Linux), in AWS your EC2 can have multiple EBS volumes.

Example: You have an EC2 (server) with one default disk (root). If you need more storage, you add another EBS volume (like plugging in an external HDD).


What we did

Attached an EBS volume → Like plugging in an extra hard disk to your EC2.

Stored data on it → Created files inside the mounted EBS volume.

Took a Snapshot → Snapshot = A backup photo of the disk at that moment (like taking a WhatsApp screenshot of your storage).

Restored Snapshot → Created a new EBS volume from that snapshot and attached it to another EC2 instance. The new instance immediately got access to the same files.

Exported Snapshot to S3 (optional) → Sent the backup to S3 (Amazon’s storage service, like Google Drive) so it’s safe, portable, and can even be used outside AWS.


📌 Example to understand

Imagine your personal laptop:

You plug in an external HDD (EBS).

You save your photos on it.

You take a backup copy on Google Drive (S3).

Later, if your laptop dies, you can plug that HDD or download the backup from Google Drive to a new laptop.

That’s exactly what we did with AWS EC2 + EBS + S3.

1. Check attached disks
lsblk

👉 Shows all disks attached to your server (like Disk Management in Windows).

2. Format the new volume
sudo mkfs -t ext4 /dev/xvdf


👉 Creates a filesystem (ext4) on the new disk so it can store files (like formatting a pen drive before use).

3. Create a folder to mount
sudo mkdir /data


👉 Makes a new folder /data where your EBS volume will be accessible.

4. Mount the volume
sudo mount /dev/xvdf /data
👉 Connects your new EBS disk (/dev/xvdf) to the folder /data.
Now when you go to /data, you are actually inside the EBS volume.


5. Make mount permanent
echo "/dev/xvdf /data ext4 defaults,nofail 0 2" | sudo tee -a /etc/fstab

👉 Ensures the disk auto-mounts every time the server restarts (otherwise you’d have to mount manually again).

6. Write files
echo "Hello Backup Project" > file1.txt
echo "This is EBS snapshot test" > file2.txt

👉 Creates two files (file1.txt and file2.txt) with the text inside them.
If you see "Permission denied", it’s because the folder is owned by root → fix with:

sudo chown ec2-user:ec2-user /data (or your username)

🔹 AWS CLI Commands
7. Create Snapshot (backup)

aws ec2 create-snapshot --volume-id vol-xxxxxxxx --description "Backup snapshot"

👉 Takes a backup (snapshot) of your EBS volume and stores it in AWS.
Equivalent to: “Save a copy of my hard disk right now.”

8. Create new volume from snapshot (restore)
aws ec2 create-volume --snapshot-id snap-xxxxxxxx --availability-zone us-east-1a

9. Export snapshot to S3
aws ec2 export-snapshot \
  --snapshot-id snap-xxxxxxxx \
  --destination-region us-east-1 \
  --s3-bucket my-backup-bucket \
  --disk-image-format VMDK

👉 Converts the snapshot into a virtual disk file (VMDK/VHD/RAW) and stores it in your S3 bucket.
Equivalent to: “Upload my backup copy to Google Drive (S3) so I can download it anytime.”

🔹 Quick Analogy

EBS volume → External hard disk.

Mounting → Plugging the disk into your PC.

Snapshot → Taking a photo of the disk’s content.

Restore → Making a new disk from that photo.

Export to S3 → Uploading the backup to cloud storage (Google Drive equivalent).
