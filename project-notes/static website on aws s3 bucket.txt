1️⃣ What is Static Website Hosting?

A static website is a website made of files that don’t change dynamically.

Example: HTML, CSS, and images that are always the same for every user.

No server-side code like PHP, Python, or Node.js is running on the backend.

Static hosting means you store these files somewhere so people can access them over the internet.

Example: Your index.html and style.css files are stored somewhere online, and when someone visits your URL, the browser simply downloads and displays them.

2️⃣ Why AWS S3 for Static Hosting?

AWS S3 (Simple Storage Service) is a service designed to store files (objects) in the cloud.

Reasons we use S3 for static websites:

Simple & Reliable: S3 stores files safely and is always available.

Cost-effective: You only pay for storage and bandwidth. Hosting a simple static website is very cheap.

Easy Public Access: You can make your files publicly accessible so everyone can see them.

Scalable: Can handle thousands of visitors without setting up servers.

Integrates with CDN (CloudFront): Makes your website load faster globally.

Example analogy:
Think of S3 as a digital bookshelf. You put your index.html and style.css files on the shelf, and anyone with a link can open the book. You don’t need a librarian (server) to serve the book every time—it’s ready to read.

3️⃣ Purpose of Using S3 in This Project

We needed a place to host our static website files.

S3 allows us to upload the HTML/CSS files and make them accessible on the internet.

By enabling static website hosting and public access, anyone can visit our site via a URL.

Optional: CloudFront acts like a delivery service, sending files from servers closer to the visitor, making it faster.
